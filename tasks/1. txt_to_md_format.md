# Implementation Plan: “Transcript to Markdown” Obsidian Plugin

## 1. Goal and Scope

- Create an Obsidian community plugin called **Transcript to Markdown**.
- Input: existing `.txt` or `.vtt` transcript files stored *inside* the vault.
- Output: `.md` notes with cleaned transcript content, stored in a configurable vault folder (default: `Transcripts`).
- Trigger: command in Command Palette (and optional hotkey) when a `.txt` or `.vtt` file is the active file.

***

## 2. Environment Setup

### 2.1. Prerequisites

- Node.js ≥ 16 installed (`node --version`).[^2]
- Git installed (for cloning template).
- Editor: VS Code or similar with TypeScript support.


### 2.2. Create project from sample plugin

1. Clone the official sample plugin template.[^3][^2]

```bash
git clone https://github.com/obsidianmd/obsidian-sample-plugin zoom-transcript-to-md
cd zoom-transcript-to-md
```

2. Install dependencies.[^2]

```bash
npm install
```

3. Optional: run dev watcher while developing.

```bash
npm run dev
```


***

## 3. Project Structure and Manifest

### 3.1. Files to customize

- `manifest.json` – plugin metadata and basic config.[^1]
- `main.ts` – main plugin logic and commands.[^4]
- `styles.css` – unused initially (can be empty).
- `versions.json` – for release later (optional for local use).[^5][^6]


### 3.2. Configure `manifest.json`

1. Open `manifest.json` and edit fields to something like:[^1]

```json
{
  "id": "zoom-transcript-to-md",
  "name": "Transcript to Markdown",
  "version": "0.1.0",
  "minAppVersion": "1.5.0",
  "author": "Your Name",
  "description": "Convert .txt/.vtt transcript files to Markdown and store them in a specific folder.",
  "authorUrl": "",
  "isDesktopOnly": true
}
```

2. Ensure:
    - `id` is unique in your vault and does not contain `obsidian`.[^1]
    - `isDesktopOnly` is `true` because plugin uses Node/Electron filesystem APIs.[^1]

## 4. Plugin Logic Design

### 4.1. High-level behavior

- When user runs the command:
    - Get currently active file via `app.workspace.getActiveFile()`.
    - Check that:
        - File exists.
        - Extension is `.txt` or `.vtt`.
    - Read file contents via `app.vault.read(file)`.
    - Depending on extension:
        - `.txt` → `convertTxtToMarkdown(content)`.
        - `.vtt` → `convertVttToMarkdown(content)`.
    - Ensure target output folder exists (create if needed).

```
- Build output path: `<outputFolder>/<basename>.md`.  
```

    - If file exists at that path, overwrite contents; otherwise, create new file.
    - Show `Notice` indicating success or error.


### 4.2. Settings

- Plugin settings interface `TranscriptPluginSettings`:
    - `outputFolder: string` – path relative to vault root (default: `"Transcripts"`).
- Settings stored via `loadData()` / `saveData()` in plugin class.
- Settings tab lets user change output folder.

## 5. Implementation in `main.ts`

### 5.1. Imports and settings types

1. Replace sample plugin content with imports and settings:

```ts
import {
  App,
  Notice,
  Plugin,
  PluginSettingTab,
  Setting,
  TFile,
  TFolder,
  normalizePath
} from "obsidian";

interface TranscriptPluginSettings {
  outputFolder: string;
}

const DEFAULT_SETTINGS: TranscriptPluginSettings = {
  outputFolder: "Transcripts"
};
```


### 5.2. Main plugin class

2. Implement plugin skeleton with lifecycle and command:

```ts
export default class TranscriptToMdPlugin extends Plugin {
  settings: TranscriptPluginSettings;

  async onload() {
    await this.loadSettings();

    this.addCommand({
      id: "convert-transcript-file",
      name: "Convert transcript file (txt/vtt) to Markdown",
      checkCallback: (checking) => {
        const file = this.app.workspace.getActiveFile();
        if (!file) return false;

        if (!file.path.endsWith(".txt") && !file.path.endsWith(".vtt")) {
          return false;
        }

        if (!checking) {
          this.convertTranscript(file);
        }

        return true;
      }
    });

    this.addSettingTab(new TranscriptSettingTab(this.app, this));
  }

  async convertTranscript(file: TFile) {
    try {
      const content = await this.app.vault.read(file);

      const isVtt = file.path.endsWith(".vtt");
      const markdown = isVtt
        ? convertVttToMarkdown(content)
        : convertTxtToMarkdown(content);

      const outputFolderPath = normalizePath(this.settings.outputFolder);

      let folder = this.app.vault.getAbstractFileByPath(
        outputFolderPath
      ) as TFolder | null;

      if (!folder) {
        folder = await this.app.vault.createFolder(outputFolderPath);
      }

      const baseName = file.basename;
      const outputPath = normalizePath(
        `${outputFolderPath}/${baseName}.md`
      );

      const existing = this.app.vault.getAbstractFileByPath(
        outputPath
      ) as TFile | null;

      if (existing) {
        await this.app.vault.modify(existing, markdown);
      } else {
        await this.app.vault.create(outputPath, markdown);
      }

      new Notice(`Transcript converted: ${outputPath}`);
    } catch (e) {
      console.error(e);
      new Notice("Error converting transcript, see console for details");
    }
  }

  async loadSettings() {
    this.settings = Object.assign(
      {},
      DEFAULT_SETTINGS,
      await this.loadData()
    );
  }

  async saveSettings() {
    await this.saveData(this.settings);
  }
}
```


### 5.3. Settings tab implementation

3. Implement `TranscriptSettingTab` to configure output folder:

```ts
class TranscriptSettingTab extends PluginSettingTab {
  plugin: TranscriptToMdPlugin;

  constructor(app: App, plugin: TranscriptToMdPlugin) {
    super(app, plugin);
    this.plugin = plugin;
  }

  display(): void {
    const { containerEl } = this;
    containerEl.empty();

    containerEl.createEl("h2", {
      text: "Transcript to Markdown Settings"
    });

    new Setting(containerEl)
      .setName("Output folder")
      .setDesc("Folder in the vault where converted Markdown files are stored")
      .addText((text) =>
        text
          .setPlaceholder("Transcripts")
          .setValue(this.plugin.settings.outputFolder)
          .onChange(async (value) => {
            this.plugin.settings.outputFolder = value || "Transcripts";
            await this.plugin.saveSettings();
          })
      );
  }
}
```



### 6.2. `.vtt` → Markdown

- Basic goals:
    - Ignore header line (`WEBVTT`).
    - Ignore numeric cue indices.
    - Ignore timestamp lines.
    - Merge text lines for each cue into a single bullet line.

```ts
function convertVttToMarkdown(content: string): string {
  const lines = content.split(/\r?\n/);

  const blocks: { text: string[] }[] = [];
  let current: { text: string[] } | null = null;

  for (const line of lines) {
    const trimmed = line.trim();
    if (!trimmed) continue;

    if (/^WEBVTT/i.test(trimmed)) continue;
    if (/^\d+$/.test(trimmed)) continue;
    if (/^\d{2}:\d{2}:\d{2}\.\d{3} -->/.test(trimmed)) {
      if (current) blocks.push(current);
      current = { text: [] };
      continue;
    }

    if (!current) {
      current = { text: [] };
    }

    current.text.push(trimmed);
  }

  if (current) blocks.push(current);

  const mdLines: string[] = [];
  for (const block of blocks) {
    const text = block.text.join(" ");
    mdLines.push(`- ${text}`);
  }

  return mdLines.join("\n");
}
```

- Future improvements:
    - Parse speaker names if present (e.g., `Speaker 1: text`).
    - Include timestamps in parentheses or as headings.
    - Add options in settings to toggle timestamps and bullets.

## 7. Build, Install, and Test in Obsidian

### 7.1. Build the plugin

- In the project folder:

```bash
npm run build
```

- This compiles `main.ts` to `main.js` using the project’s TypeScript config.[^4][^2]


### 7.2. Manual installation into a test vault

1. Choose or create a test Obsidian vault (recommended) for development.[^4]
2. Inside the vault, create plugin folder:
    - `<vault>/.obsidian/plugins/zoom-transcript-to-md/`
3. Copy required files into that folder:[^7][^4]
    - `main.js`
    - `manifest.json`
    - `styles.css` (can be empty)

### 7.3. Enable plugin

1. In Obsidian: Settings → Community plugins → turn off Safe mode (if needed).
2. Click “Browse” to make sure community plugins are enabled.[^8]
3. Under “Installed plugins”, enable **Transcript to Markdown**.

### 7.4. Functional tests

1. Put a `.txt` transcript file in the vault, open it in the editor.
2. Press `Ctrl/Cmd+P` → run “Convert transcript file (txt/vtt) to Markdown”.
3. Verify:
    - A `Transcripts` folder (or your configured folder) is created.
    - A `.md` file with the same base name is created or updated.
    - Content is correctly converted (raw for `.txt`, bulleted for `.vtt`).
4. Repeat with `.vtt` file that has:
    - `WEBVTT` header.
    - Line numbers.
    - Timestamp lines.
    - Multi-line caption blocks.
5. Confirm that invalid cases (no active file, wrong extension) simply do not expose the command (or do nothing), thanks to `checkCallback` guard.
