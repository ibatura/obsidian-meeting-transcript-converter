# Auto‑Convert Transcript Files Spec

## 1. Goal

Automatically convert newly added `.txt` and `.vtt` transcript files in a chosen vault folder to Markdown, with configurable:

- Watched folder (where new transcripts appear).
- Output folder (where `.md` files are created).
- Auto‑convert on/off.
- Option to delete the original `.txt`/`.vtt` after successful conversion.

***

## 2. Settings

Extend plugin settings:

```ts
interface TranscriptPluginSettings {
  outputFolder: string;                // Folder where MD notes are stored
  watchFolder: string;                 // Folder to watch for new txt/vtt
  autoConvertEnabled: boolean;         // Enable/disable auto-conversion
  deleteOriginalAfterConvert: boolean; // Remove source file after convert
}

const DEFAULT_SETTINGS: TranscriptPluginSettings = {
  outputFolder: "Transcripts",
  watchFolder: "Transcripts", // or "" = whole vault
  autoConvertEnabled: false,
  deleteOriginalAfterConvert: false
};
```

Behaviour:

- `outputFolder` and `watchFolder` are paths **relative to vault root**.
- `watchFolder = ""` → watch **entire** vault.
- `outputFolder` and `watchFolder` can be the **same folder**.
- `deleteOriginalAfterConvert = true` → delete `.txt`/`.vtt` only after the `.md` file is created without error.

## 3. Settings Tab UI

In `TranscriptSettingTab.display()`:

```ts
display(): void {
  const { containerEl } = this;
  containerEl.empty();

  containerEl.createEl("h2", {
    text: "Transcript to Markdown Settings"
  });

  // Output folder
  new Setting(containerEl)
    .setName("Output folder")
    .setDesc("Folder where converted Markdown files are stored")
    .addText(text =>
      text
        .setPlaceholder("Transcripts")
        .setValue(this.plugin.settings.outputFolder)
        .onChange(async (value) => {
          this.plugin.settings.outputFolder = value || "Transcripts";
          await this.plugin.saveSettings();
        })
    );

  // Watch folder
  new Setting(containerEl)
    .setName("Watch folder")
    .setDesc("Folder to watch for new .txt/.vtt files (empty = whole vault)")
    .addText(text =>
      text
        .setPlaceholder("Transcripts")
        .setValue(this.plugin.settings.watchFolder)
        .onChange(async (value) => {
          this.plugin.settings.watchFolder = value.trim();
          await this.plugin.saveSettings();
        })
    );

  // Auto-convert toggle
  new Setting(containerEl)
    .setName("Auto-convert new transcripts")
    .setDesc("Automatically convert new .txt/.vtt files in the watched folder")
    .addToggle(toggle =>
      toggle
        .setValue(this.plugin.settings.autoConvertEnabled)
        .onChange(async (value) => {
          this.plugin.settings.autoConvertEnabled = value;
          await this.plugin.saveSettings();
        })
    );

  // Delete original toggle
  new Setting(containerEl)
    .setName("Delete original file after convert")
    .setDesc("Remove the source .txt/.vtt file after the .md file is created")
    .addToggle(toggle =>
      toggle
        .setValue(this.plugin.settings.deleteOriginalAfterConvert)
        .onChange(async (value) => {
          this.plugin.settings.deleteOriginalAfterConvert = value;
          await this.plugin.saveSettings();
        })
    );
}
```

## 4. File Create Watcher

Use Obsidian’s vault `create` event, registered in `onload()` and cleaned automatically on unload via `registerEvent`.[^3][^4]

In the plugin class:

```ts
async onload() {
  await this.loadSettings();

  // Existing command and settings tab registration...

  this.addCommand({
    id: "convert-transcript-file",
    name: "Convert transcript file (txt/vtt) to Markdown",
    checkCallback: (checking) => {
      const file = this.app.workspace.getActiveFile();
      if (!file) return false;
      if (!file.path.endsWith(".txt") && !file.path.endsWith(".vtt")) {
        return false;
      }
      if (!checking) {
        this.convertTranscript(file);
      }
      return true;
    }
  });

  this.addSettingTab(new TranscriptSettingTab(this.app, this));

  // Watch new files in the vault
  this.registerEvent(
    this.app.vault.on("create", async (abstractFile) => {
      await this.handleFileCreate(abstractFile);
    })
  );
}
```

Handler implementation:

```ts
private async handleFileCreate(abstractFile: TAbstractFile) {
  // Auto-convert must be enabled in settings
  if (!this.settings.autoConvertEnabled) return;

  // Only act on files, not folders
  if (!(abstractFile instanceof TFile)) return;

  const file = abstractFile;

  // Only .txt or .vtt
  if (!file.path.endsWith(".txt") && !file.path.endsWith(".vtt")) return;

  // If watchFolder is set, restrict to that folder subtree
  const watch = this.settings.watchFolder?.trim();
  if (watch) {
    const normalizedWatch = normalizePath(watch) + "/";
    const normalizedPath = normalizePath(file.path);
    if (!normalizedPath.startsWith(normalizedWatch)) {
      return;
    }
  }

  // Convert file to Markdown
  await this.convertTranscript(file, false);

  // Optional deletion of original file
  if (this.settings.deleteOriginalAfterConvert) {
    try {
      await this.app.vault.delete(file);
    } catch (e) {
      console.error(e);
      new Notice("Failed to delete original transcript file");
    }
  }

  new Notice(`Auto-converted transcript: ${file.path}`);
}
```

Notes:

- `watchFolder` is treated as a path prefix; `"Inbox"` matches `"Inbox/file.vtt"` and `"Inbox/sub/file.txt"`.
- If `watchFolder` is empty, **any** new `.txt`/`.vtt` anywhere in the vault is processed.
- Deletion only happens after `convertTranscript` returns without throwing.

## 5. Reusing `convertTranscript`

Adjust existing `convertTranscript` so it can be used from both:

- Manual command (Command Palette).
- Automatic watcher (file create event).

```ts
async convertTranscript(file: TFile, showNotice: boolean = true) {
  try {
    const content = await this.app.vault.read(file);
    const isVtt = file.path.endsWith(".vtt");
    const markdown = isVtt
      ? convertVttToMarkdown(content)
      : convertTxtToMarkdown(content);

    const outputFolderPath = normalizePath(this.settings.outputFolder);

    let folder = this.app.vault.getAbstractFileByPath(
      outputFolderPath
    ) as TFolder | null;

    if (!folder) {
      folder = await this.app.vault.createFolder(outputFolderPath);
    }

    const baseName = file.basename;
    const outputPath = normalizePath(
      `${outputFolderPath}/${baseName}.md`
    );

    const existing = this.app.vault.getAbstractFileByPath(
      outputPath
    ) as TFile | null;

    if (existing) {
      await this.app.vault.modify(existing, markdown);
    } else {
      await this.app.vault.create(outputPath, markdown);
    }

    if (showNotice) {
      new Notice(`Transcript converted: ${outputPath}`);
    }
  } catch (e) {
    console.error(e);
    new Notice("Error converting transcript, see console for details");
  }
}
```

Manual command continues to call:

```ts
this.convertTranscript(file); // showNotice defaults to true
```

Watcher calls:

```ts
await this.convertTranscript(file, false);
```


***

## 6. Behaviour Summary

- Drop `.txt` or `.vtt` files into the **watched folder** inside the vault.
- If `autoConvertEnabled` is `true`, the plugin:
    - Filters for `.txt`/`.vtt`.
    - Checks path starts with `watchFolder` (unless empty).
    - Converts content to Markdown using existing converters.
    - Writes `.md` note into `outputFolder`.
    - Optionally deletes the original `.txt`/`.vtt` based on `deleteOriginalAfterConvert`.
    - Shows a short notice that auto‑conversion occurred.
