# Frontmatter Properties & Full Date‑Time in Dialog

## 1. Goal

Enhance the converted `.md` output with:

1. **YAML frontmatter (properties)** — structured metadata at the top of every converted note.
2. **Full date‑time in dialog lines** — each timestamp shows `YYYY-MM-DD HH:mm:ss` instead of time‑only.

***

## 2. Frontmatter Properties

Every converted note must start with a YAML frontmatter block containing the properties listed below.

### 2.1. Property Definitions

| Property       | Type     | Source / Logic                                                                                                                                                   | Example                              |
| -------------- | -------- | ---------------------------------------------------------------------------------------------------------------------------------------------------------------- | ------------------------------------ |
| `meeting_name` | Text     | Derived from the **source filename**. If the filename equals `meeting_saved_closed_caption` (case-insensitive), set to `"Untitled Meeting"`. Otherwise, take the filename (without extension), replace underscores with spaces, and use title case. | `"Weekly Sync"` or `"Untitled Meeting"` |
| `date`         | DateTime | The **file creation time** (`file.stat.ctime`) formatted as `YYYY-MM-DD HH:mm:ss`.                                                                              | `2026-04-05 14:30:00`               |
| `duration`     | Text     | Calculated from timestamps in the transcript. **VTT:** last cue's end timestamp. **TXT:** difference between the last and first `[Speaker] HH:mm:ss` timestamps. Format: `HH:mm:ss`. If no timestamps found, omit. | `"01:23:45"`                         |
| `participants` | List     | Extract unique speaker names. **VTT:** from content lines matching `Name: text`. **TXT:** from header lines matching `[Speaker Name] HH:mm:ss`. If no speakers detected, omit this property. | `["Alice", "Bob"]`                   |

### 2.2. Output Format

```yaml
---
meeting_name: Weekly Sync
date: 2026-04-05 14:30:00
duration: "01:23:45"
participants:
  - Alice
  - Bob
---
```

### 2.3. Meeting Name Logic (Pseudocode)

```
basename = file.basename               // e.g. "meeting_saved_closed_caption"
normalized = basename.toLowerCase()

if normalized == "meeting_saved_closed_caption":
    meetingName = "Untitled Meeting"
else:
    meetingName = basename
        .replace(/_/g, " ")
        .replace(/\b\w/g, c => c.toUpperCase())   // title case
```

### 2.4. Duration Logic

Duration is calculated for **both** file types.

#### VTT Duration

```
Parse all timestamp lines matching:  START --> END
Track the maximum END timestamp across all cues.
Convert maximum END offset (ms) to HH:mm:ss string.
```

#### TXT Duration

TXT transcripts have header lines in the format `[Speaker Name] HH:mm:ss`.

```
Collect all timestamps from lines matching: /^\[.+?\]\s+(\d{2}:\d{2}:\d{2})/
firstTime = first matched timestamp
lastTime  = last matched timestamp
duration  = lastTime - firstTime
Format as HH:mm:ss.
```

> **Note:** TXT timestamps are wall-clock times (e.g. `20:30:11`), not offsets. Duration is simply `lastTime − firstTime`.

### 2.5. Participants Extraction Logic

Participants are extracted differently depending on file type.

#### VTT Participants

```
For each content line (non-timestamp, non-WEBVTT, non-sequence):
    match = line.match(/^(.+?):\s/)
    if match:
        add match[1].trim() to participants set

Return unique list, sorted alphabetically.
```

> **Note:** To reduce false positives (e.g. `"Next steps: review the plan"`), ignore matches where the candidate name is longer than 40 characters or contains common words like `http`, `https`, `Note`, `TODO`.

#### TXT Participants

TXT transcripts use the format `[Speaker Name] HH:mm:ss` on header lines.

```
For each line matching /^\[(.+?)\]\s+\d{2}:\d{2}:\d{2}/:
    add captured group (speaker name) to participants set

Return unique list, sorted alphabetically.
```

## 3. Full Date‑Time in Dialog Lines

### 3.1. Current Behaviour

**VTT:** The converter computes an absolute timestamp per cue using `moment(fileCreationTime + block.timeOffset).format(timeFormat)`. With `timeFormat = "HH:mm:ss DD:MM:YYYY"`, it already includes a date but the format is inconsistent.

**TXT:** The current `convertTxtToMarkdown` does no timestamp processing at all — it simply trims and joins lines. The TXT format has header lines like:

```
[Yulia Vovk] 20:30:11
Hello everyone, welcome to the meeting.
[Ivan Batura] 20:30:25
Thanks, let's get started.
```

Each header line contains `[Speaker Name] HH:mm:ss` (time-only), followed by the message on the next line.

### 3.2. Required Changes

#### VTT

- Change the **default** `timeFormat` in `DEFAULT_SETTINGS` to:

  ```ts
  timeFormat: "YYYY-MM-DD HH:mm:ss"
  ```

- The dialog line format stays:

  ```
  - **[2026-04-05 14:31:05]** Hello everyone, let's get started.
  ```

#### TXT

- The TXT converter must now **parse** header lines matching `[Speaker Name] HH:mm:ss`.
- **Prepend the date** (derived from `file.stat.ctime`) to the existing time, so `[Yulia Vovk] 20:30:11` becomes `[Yulia Vovk] 2026-04-05 20:30:11`.
- The date portion uses the format `YYYY-MM-DD` (always, not configurable separately — only the date comes from `file.stat.ctime`).
- The original two-line structure is **preserved** — the header line keeps the `[Speaker Name] datetime` format, followed by the message on the next line:

  ```
  [Yulia Vovk] 2026-04-05 20:30:11
  Hello everyone, welcome to the meeting.
  ```

- The `convertTxtToMarkdown` function signature must change to accept `timeFormat` and `fileCreationTime` (same as `convertVttToMarkdown`).

### 3.3. TXT Conversion Logic (Updated)

```
datePrefix = moment(fileCreationTime).format("YYYY-MM-DD")

For each line:
    if line matches /^\[(.+?)\]\s+(\d{2}:\d{2}:\d{2})/:
        speaker = match[1]
        time    = match[2]
        replace line with: [{speaker}] {datePrefix} {time}
    else:
        keep line as-is (message lines, empty lines, etc.)
```

### 3.4. Settings Tab Description Update

Update the `timeFormat` setting description to:

> `"Timestamp format for dialog lines (moment.js). Default: YYYY-MM-DD HH:mm:ss"`

### 4.2. New Utility Functions

#### `extractParticipants(content: string, format: "vtt" | "txt"): string[]`

```ts
export function extractParticipants(content: string, format: "vtt" | "txt"): string[] {
    const speakers = new Set<string>();
    const lines = content.split(/\r?\n/);

    if (format === "txt") {
        // TXT format: [Speaker Name] HH:mm:ss
        for (const line of lines) {
            const match = line.trim().match(/^\[(.+?)\]\s+\d{2}:\d{2}:\d{2}/);
            if (match && match[1]) {
                speakers.add(match[1].trim());
            }
        }
    } else {
        // VTT format: "Speaker Name: message text"
        const ignore = /^(http|https|note|todo)/i;
        for (const line of lines) {
            const trimmed = line.trim();
            if (!trimmed) continue;
            if (/^WEBVTT/i.test(trimmed)) continue;
            if (/^\d+$/.test(trimmed)) continue;
            if (/-->/.test(trimmed)) continue;

            const match = trimmed.match(/^(.+?):\s/);
            if (match && match[1]) {
                const name = match[1].trim();
                if (name.length <= 40 && !ignore.test(name)) {
                    speakers.add(name);
                }
            }
        }
    }

    return Array.from(speakers).sort();
}
```

#### `extractDuration(content: string, format: "vtt" | "txt"): string | null`

```ts
export function extractDuration(content: string, format: "vtt" | "txt"): string | null {
    const lines = content.split(/\r?\n/);

    if (format === "txt") {
        // TXT: collect wall-clock times from [Speaker] HH:mm:ss lines
        const times: number[] = [];
        for (const line of lines) {
            const match = line.trim().match(/^\[.+?\]\s+(\d{2}):(\d{2}):(\d{2})/);
            if (match) {
                const secs = parseInt(match[1], 10) * 3600
                           + parseInt(match[2], 10) * 60
                           + parseInt(match[3], 10);
                times.push(secs);
            }
        }
        if (times.length < 2) return null;

        const first = times[0];
        const last  = times[times.length - 1];
        let diff = last - first;
        // Handle midnight crossing (e.g. 23:55:00 → 00:05:00)
        if (diff < 0) diff += 24 * 3600;

        const h = Math.floor(diff / 3600);
        const m = Math.floor((diff % 3600) / 60);
        const s = diff % 60;
        return `${String(h).padStart(2, "0")}:${String(m).padStart(2, "0")}:${String(s).padStart(2, "0")}`;
    }

    // VTT: find the maximum end timestamp
    let maxMs = 0;
    for (const line of lines) {
        const trimmed = line.trim();
        const timeMatch = trimmed.match(
            /-->\s*(\d{2}:\d{2}:\d{2}\.\d{3}|\d{2}:\d{2}\.\d{3})/
        );
        if (timeMatch && timeMatch[1]) {
            const ms = parseVttTimeOffset(timeMatch[1]);
            if (ms > maxMs) maxMs = ms;
        }
    }

    if (maxMs === 0) return null;

    const totalSeconds = Math.floor(maxMs / 1000);
    const h = Math.floor(totalSeconds / 3600);
    const m = Math.floor((totalSeconds % 3600) / 60);
    const s = totalSeconds % 60;
    return `${String(h).padStart(2, "0")}:${String(m).padStart(2, "0")}:${String(s).padStart(2, "0")}`;
}
```

> **Note:** `parseVttTimeOffset` is currently a private (non-exported) function in `converters.ts`. It must be exported so `extractDuration` can reuse it (or keep both in the same file scope).

#### Updated `convertTxtToMarkdown(content: string, timeFormat: string, fileCreationTime: number): string`

The TXT converter now parses `[Speaker] HH:mm:ss` header lines and prepends the date. The original two-line structure (header + message) is preserved.

```ts
export function convertTxtToMarkdown(content: string, timeFormat: string, fileCreationTime: number): string {
    const lines = content.split(/\r?\n/);
    const mdLines: string[] = [];
    const datePrefix = moment(fileCreationTime).format("YYYY-MM-DD");

    for (const line of lines) {
        const trimmed = line.trim();
        if (trimmed === "") continue;

        // Match header line: [Speaker Name] HH:mm:ss
        const headerMatch = trimmed.match(/^\[(.+?)\]\s+(\d{2}:\d{2}:\d{2})/);
        if (headerMatch) {
            const speaker = headerMatch[1];
            const time = headerMatch[2];
            mdLines.push(`[${speaker}] ${datePrefix} ${time}`);
        } else {
            // Message line or other content — keep as-is
            mdLines.push(trimmed);
        }
    }

    return mdLines.join("\n");
}
```

### 4.3. Frontmatter Builder in `convertTranscript.ts`

```ts
// After reading file content and before generating mdContent:

const format = file.extension === "vtt" ? "vtt" : "txt";
const meetingName = deriveMeetingName(file.basename);
const dateStr = moment(fileCreationTime).format("YYYY-MM-DD HH:mm:ss");

let frontmatter = `---\nmeeting_name: "${meetingName}"\ndate: ${dateStr}\n`;

const duration = extractDuration(content, format);
if (duration) {
    frontmatter += `duration: "${duration}"\n`;
}

const participants = extractParticipants(content, format);
if (participants.length > 0) {
    frontmatter += `participants:\n`;
    for (const p of participants) {
        frontmatter += `  - "${p}"\n`;
    }
}

frontmatter += `---\n\n`;

// Prepend frontmatter to mdContent
mdContent = `${frontmatter}# ${title}\n\n${mdContent}`;
```

#### `deriveMeetingName`

```ts
function deriveMeetingName(basename: string): string {
    if (basename.toLowerCase() === "meeting_saved_closed_caption") {
        return "Untitled Meeting";
    }
    return basename
        .replace(/_/g, " ")
        .replace(/\b\w/g, (c) => c.toUpperCase());
}
```

## 7. Behaviour Summary

- Every converted `.md` note now begins with YAML frontmatter containing `meeting_name`, `date`, and optionally `duration` and `participants`.
- **Duration** is calculated for **both** VTT and TXT files: VTT uses the last cue's end timestamp; TXT uses the difference between the first and last `[Speaker] HH:mm:ss` timestamps.
- **Participants** are extracted from both formats: VTT uses `Name: text` patterns; TXT uses `[Name] HH:mm:ss` header lines.
- The `meeting_name` is smart: generic filenames like `meeting_saved_closed_caption` produce `"Untitled Meeting"`, while descriptive filenames become title-cased names.
- **TXT dialog lines** now have the date prepended: `[Speaker Name] HH:mm:ss` becomes `[Speaker Name] YYYY-MM-DD HH:mm:ss`. The original two-line structure (header + message) is preserved.
- **VTT dialog timestamps** now show full date-time (`YYYY-MM-DD HH:mm:ss`) by default.
- The `convertTxtToMarkdown` function signature changes to accept `timeFormat` and `fileCreationTime` (breaking change for callers).
- All changes are backward-compatible for settings — existing user settings are preserved, and the time format remains configurable.
